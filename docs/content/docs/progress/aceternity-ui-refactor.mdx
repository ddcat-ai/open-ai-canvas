# Aceternity UI 重构计划

本次工作是产品展示层和交互层重构，不是为旧组件补充局部动画。业务数据、API、store、service 和已验证的业务 hook 保持稳定；组件结构、视觉表面、动效和页面编排按 Aceternity-first 原则重写。

## 不可回退原则

- Aceternity UI 是核心产品界面的主视觉与交互参考。
- 同一入口不长期维护新旧两套视觉实现；新组件接管后删除旧分支、旧配置和重复样式。
- 外部组件源码必须适配当前命令、状态、主题和中文交互，不照搬营销 demo 的 `href`、硬编码颜色或 Tabler 图标。
- 高密度画布可以调整透视、位移和弹簧参数，但不能退回静态平面按钮或卡片。
- `prefers-reduced-motion`、触摸端和低缩放属于明确的交互状态，不是旧 UI 兜底入口。

## 目标组件层

- `components/ui/aceternity/`：跨页面使用的 Floating Dock、Comet Card、Gooey Input、File Upload、Background Lines、Animated Modal、Stateful Button 和结果切换原语。
- `components/canvas/`：结合画布事件和业务状态的节点表面、上传覆盖层、节点 Dock、生成反馈和屏幕空间浮层。
- `lib/aceternity-motion.ts`：统一时长、弹簧和缓动；颜色与空间表面继续由 `canvasThemes` 提供。

## 每阶段验收边界

- 节点定位、拖动、缩放、连线、编辑和生成语义不得变化。
- 明暗主题、键盘焦点、触摸端和 reduced-motion 必须有明确状态。
- 高频指针值使用 MotionValue、ref 或直接 DOM 更新，不通过 React state 逐帧重渲染。
- 发现位置漂移、连线错位、面板遮挡或明显掉帧时，修正组件所有权和运动参数，不恢复旧视觉分支。
- 按项目规则不执行语法检查、测试或构建；已实现变化先登记到待测试清单，由人工确认后更新正式功能说明。
