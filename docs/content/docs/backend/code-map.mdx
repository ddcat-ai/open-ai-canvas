## 后端调用链

```text
HTTP 请求 -> handler -> service -> repository/model -> SQLite/PostgreSQL 或资源存储
                         |
                         -> provider/outbound -> 模型上游
```

| 功能 | Handler | Service / Repository |
| --- | --- | --- |
| 路由与中间件 | `handler/routes.go` | `service/auth.go`、`handler/security.go` |
| 用户和管理员 | `handler/auth.go` | `service/auth.go`、`service/admin.go` |
| 管理后台批量操作 | `handler/auth.go` | `service/admin.go`、`repository/admin_audit.go`、`repository/analytics.go` |
| 异步任务与 Agent | `handler/routes.go` | `service/service.go`、`service/provider.go` |
| 用户画布与素材 | `handler/user_data.go` | `service/user_data.go` |
| 文件和 OSS | `handler/routes.go` | `service/resource.go`、`service/settings.go` |
| 画布分享 | `handler/canvas_share.go` | `service/canvas_share.go` |
| 技能库 | `handler/skills.go` | `service/skills.go` |
| 用量分析 | `handler/admin_analytics.go` | `service/analytics.go`、`repository/analytics.go` |

## 其他入口

- `canvas-agent/src/`：本地 Agent、MCP 工具和当前画布会话桥接。
- `plugins/infinite-canvas/`：Codex App 插件清单与技能。
- `docs/content/docs/`：产品、操作、开发和进度文档。
- `docker-compose.local.yml`：本地前后端编排；`nginx.conf`：生产静态资源和 API 代理。

## 修改原则

- HTTP 参数与响应只在 handler 处理，业务规则进入 service。
- 页面保留流程协调；可独立理解和测试的算法进入同业务域的 `lib`。
- 新增功能时只更新本页的主要入口，不增加逐文件清单。
