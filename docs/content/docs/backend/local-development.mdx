# 本地开发

当前主应用由 `web/` 前端和 `backend/` 任务中心组成。AI 渠道配置保存在浏览器本地；画布生成、视频编辑任务和 Agent 会话会提交到 `backend/` 队列，由后端调用用户配置的 OpenAI 兼容接口。

## 1. 启动前端

```bash
cd web
bun install
bun run dev
```

前端默认访问：

```text
http://localhost:3000
```

## 2. 配置模型

打开右上角配置弹窗，填写自己的 `Base URL`、`API Key` 和模型名。生成任务会通过后端队列执行。

## 3. 启动任务中心后端

```bash
cd backend
go mod tidy
go run ./cmd/server
```

后端默认监听：

```text
http://127.0.0.1:8080
```

Vite 开发服务会把 `/api` 代理到该地址。

后端默认拒绝访问 localhost 和私网模型地址。若开发环境的模型服务也运行在本机或可信内网，需要仅在本地启动命令前设置：

```bash
CANVAS_ALLOW_PRIVATE_UPSTREAMS=true go run ./cmd/server
```

已有首个管理员后，公开注册默认关闭；本地需要继续注册测试账号时可临时设置 `CANVAS_REGISTRATION_ENABLED=true`。

## 4. 启动文档站

如果需要单独调整文档站，在 `docs` 目录执行：

```bash
bun run dev
```

## 常见场景

- 改画布、页面和交互：主要看 `web/`
- 改任务中心和 Agent 会话：主要看 `backend/`
- 改文档站内容：主要看 `docs/content/docs/`
